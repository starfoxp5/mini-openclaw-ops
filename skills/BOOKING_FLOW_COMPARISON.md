# GoBooking 預約流程比對

## 版本

- 原版：`openclaw-gobooking-booking-legacy`
- 整合版：`openclaw-gobooking-booking-integrated`
- 我的新版：`openclaw-gobooking-booking`

## 差異重點

1. 流程治理
- 原版：強調先收集欄位、先給送出前摘要、使用者確認後才送出。
- 整合版：保留原版治理，同時加入網站細節強約束。
- 我的新版：以網站操作精準性為主，治理規則較偏執行層。

2. 優惠碼
- 原版：有折扣碼步驟，但未強制完整檢查順序。
- 整合版：強制 `Search -> 出現優惠內容 -> Apply`。
- 我的新版：同整合版，且要求折扣後金額驗證。

3. 自動送出
- 原版：原則上都要再確認。
- 整合版：僅在「本次由使用者提供優惠碼 + 已 Apply + Total=0 TWD」可直接送出。
- 我的新版：同整合版。

4. 姓名辨識
- 原版：無此規則。
- 整合版：付款人姓名格式 `原姓名（Agent名）`。
- 我的新版：同整合版，含欄位長度備援。

5. 失敗備援
- 原版：要求至少提供兩個替代方案。
- 整合版：保留兩個替代方案，並加上時段失敗重試規則。
- 我的新版：有重試與錯誤恢復，但未明確繼承原版固定回覆格式。

## 建議

- 想最穩且不改團隊習慣：選「整合版」。
- 想完全維持舊行為：選「原版」。
- 想最快落地網站自動化規則：選「我的新版」。
